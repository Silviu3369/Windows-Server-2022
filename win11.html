Windows 11 - Lecții esențiale (pas cu pas)

Conținut inspirat din capitolele-cheie ale syllabusului tău, tradus pentru învățare practică.

Content inspired by key syllabus chapters, translated for practical learning.

L1. Introducere și cerințe

Teorie: Windows 11 funcționează pe modelul Windows as a Service. Alegerea ediției și verificarea compatibilității hardware influențează stabilitatea pe termen lung.

Theory: Windows 11 runs on Windows as a Service. Edition selection and hardware compatibility directly affect long-term stability.

Pași / Steps

  1. Verifică cerințe hardware și TPM.
  2. Alege ediția corectă (Pro/Enterprise).
  3. Planifică update channel.

Verificare / Validation

  • Device compatible.
  • Canalul de update documentat.
Get-ComputerInfo | Select OsName,WindowsVersion
Confirm-SecureBootUEFI
Get-Tpm

L2. Instalare și activare

Teorie: instalarea corectă include media validă, activare și baseline de updates înainte de utilizare productivă.

Theory: proper installation includes valid media, activation, and update baseline before production use.

Pași / Steps

  1. Rulează instalarea curată.
  2. Activează sistemul.
  3. Aplică update-uri critice.

Verificare / Validation

  • Windows activated.
  • No pending critical updates.
slmgr /xpr
Get-WindowsUpdateLog
Get-HotFix | Sort InstalledOn -Descending

L3. Accounts și identitate

Teorie: identitatea poate fi locală, Microsoft, AD sau Entra ID. UAC și grupurile definesc nivelul de control al utilizatorului.

Theory: identity can be local, Microsoft, AD, or Entra ID. UAC and groups define user control level.

Pași / Steps

  1. Configurează tipul de cont necesar.
  2. Setează politici de parole.
  3. Validează apartenența la grupuri.

Verificare / Validation

  • User login is consistent.
  • Rights are appropriate.
Get-LocalUser
whoami /groups
net user

L4. Storage și fișiere

Teorie: administrarea stocării include volume, NTFS permissions, ownership și politici de organizare.

Theory: storage management includes volumes, NTFS permissions, ownership, and organization policies.

Pași / Steps

  1. Verifică discuri și volume.
  2. Aplică ACL pe foldere critice.
  3. Testează accesul cu user standard.

Verificare / Validation

  • Volume healthy.
  • Permissions enforced.
Get-Disk
Get-Volume
icacls C:\Data

L5. Networking în Windows 11

Teorie: conectivitatea în rețea depinde de configurarea corectă TCP/IP, DNS și profiluri de rețea.

Theory: network connectivity depends on proper TCP/IP, DNS, and network profile configuration.

Pași / Steps

  1. Verifică IP/DNS.
  2. Testează conectivitate spre servere.
  3. Join domain dacă e cerut.

Verificare / Validation

  • DNS resolution works.
  • Domain resources reachable.
ipconfig /all
Test-NetConnection SRV-DC01
Add-Computer -DomainName academy.local -Restart

L6. Security baseline

Teorie: securitatea endpoint include Defender, Firewall, SmartScreen și update-uri regulate.

Theory: endpoint security includes Defender, Firewall, SmartScreen, and regular updates.

Pași / Steps

  1. Verifică status Defender.
  2. Activează profile firewall.
  3. Configurează scanări periodice.

Verificare / Validation

  • Protection status healthy.
  • No critical alerts.
Get-MpComputerStatus
Get-NetFirewallProfile
Set-NetFirewallProfile -Profile Domain,Private,Public -Enabled True

L7. Services și procese

Teorie: monitorizarea proceselor și serviciilor ajută la prevenirea incidentelor de performanță.

Theory: monitoring processes and services helps prevent performance incidents.

Pași / Steps

  1. Identifică servicii critice.
  2. Verifică procese cu consum ridicat.
  3. Automatizează task-uri repetitive.

Verificare / Validation

  • Critical services running.
  • Resource spikes documented.
Get-Service
Get-Process | Sort CPU -Descending | Select -First 10
Get-ScheduledTask | Select TaskName,State -First 20

L8. Recovery și continuitate

Teorie: backup, restore și recovery trebuie testate înainte de incidente reale.

Theory: backup, restore, and recovery must be tested before real incidents.

Pași / Steps

  1. Configurează opțiuni recovery.
  2. Testează restore point.
  3. Documentează procedura de reset/recovery.

Verificare / Validation

  • Restore workflow tested.
  • Recovery steps documented.
reagentc /info
wbadmin get versions
Get-ComputerRestorePoint
