00 - Inleiding

Obiectiv / Objective

Sa intelegi diferenta functionala dintre Windows Client si Windows Server si sa planifici corect baza infrastructurii.

Understand the functional difference between Windows Client and Windows Server and correctly plan infrastructure baseline.

Teorie extinsa

Windows Client este orientat catre utilizatorul final: productivitate, aplicatii de birou, interactiune directa. Windows Server este orientat catre servicii centrale: identitate (AD), politici (GPO), fisiere partajate, print, audit, securitate si disponibilitate. Intr-o infrastructura corecta, clientul consuma servicii; serverul le guverneaza. Daca inversezi aceste roluri, apar probleme de control, securitate si scalare.

Windows Client is end-user oriented: productivity, desktop apps, direct interaction. Windows Server is centralized-service oriented: identity (AD), policy (GPO), shared files, print, audit, security, and availability. In a proper infrastructure, clients consume services; servers govern them. If these roles are mixed, control, security, and scalability issues appear.

Pasi de lucru

  1. Defineste rolurile serverului (Domain Controller, File, Print, Management).
  2. Alege editie Server 2022 (Standard/Datacenter) pe baza nevoilor.
  3. Stabileste naming convention: SRV-DC01, SRV-FS01 etc.
  4. Stabileste schema IP si DNS intern.
  5. Define server roles (Domain Controller, File, Print, Management).
  6. Choose Server 2022 edition (Standard/Datacenter) by requirements.
  7. Set naming convention: SRV-DC01, SRV-FS01 etc.
  8. Set IP schema and internal DNS.

Verificare

  • Ai document de design cu roluri si adrese IP.
  • Serverul raporteaza corect versiunea si adaptorii de retea.
  • You have a design document with roles and IP addresses.
  • Server correctly reports OS version and network adapters.
Get-ComputerInfo | Select WindowsProductName,WindowsVersion,CsName
Get-NetAdapter
Get-NetIPConfiguration

01 - OpzettenDomein (Active Directory Domain Services)

Teorie extinsa

AD DS este baza de identitate a retelei. Forest-ul este limita superioara de securitate; domeniul este unitatea de administrare pentru utilizatori/computere; OU-urile sunt containere logice pentru aplicarea politicilor si delegare. Kerberos asigura autentificarea, iar DNS intern asigura localizarea serviciilor de domeniu. Fara DNS intern corect, AD DS devine instabil.

AD DS is the network identity backbone. Forest is the top security boundary; domain is the administration unit for users/computers; OUs are logical containers for policy and delegation. Kerberos provides authentication, and internal DNS provides domain service discovery. Without proper internal DNS, AD DS becomes unstable.

Implementare pas cu pas

  1. Instaleaza rolurile AD DS si DNS pe serverul tinta.
  2. Promoveaza serverul in Domain Controller nou pentru forest nou.
  3. Dupa restart, valideaza serviciile AD si DNS.
  4. Configureaza clientii sa foloseasca DNS-ul domain controller-ului.
  5. Install AD DS and DNS roles on target server.
  6. Promote server as new Domain Controller in new forest.
  7. After restart, validate AD and DNS services.
  8. Configure clients to use DC as DNS.
Install-WindowsFeature AD-Domain-Services,DNS -IncludeManagementTools
Install-ADDSForest -DomainName "academy.local" -DomainNetbiosName "ACADEMY"
Get-ADDomain
Get-ADForest
dcdiag /v

02 - Accounts

Teorie: identitatea in AD trebuie modelata pe structura organizatiei. Nu gestionezi permisiuni pe utilizatori individuali daca poti gestiona pe grupuri. Delegarea corecta inseamna operare sigura fara a distribui drepturi administrative excesive.

Theory: AD identity must be modeled on organizational structure. Do not manage permissions per individual user when group-based management is possible. Proper delegation means safe operations without excessive admin rights.

New-ADOrganizationalUnit -Name "HR" -Path "DC=academy,DC=local"
New-ADGroup -Name "GG-HR-RW" -GroupScope Global -GroupCategory Security
New-ADUser -Name "Ana Pop" -SamAccountName apop -Enabled $true -AccountPassword (ConvertTo-SecureString "P@ssw0rd!" -AsPlainText -Force)
Add-ADGroupMember -Identity "GG-HR-RW" -Members apop

03 - File Server

Teorie: securitatea pe fisiere are doua niveluri: share permissions si NTFS permissions. Efectivul final este combinat. Quota/FSRM este control de capacitate si guvernanta date.

Theory: file security has two levels: share permissions and NTFS permissions. Final effective access is combined. Quota/FSRM provides capacity control and data governance.

Install-WindowsFeature FS-FileServer,FS-Resource-Manager -IncludeManagementTools
New-Item -Path C:\Shares\Finance -ItemType Directory
New-SmbShare -Name "Finance$" -Path C:\Shares\Finance -FullAccess "Domain Admins"
icacls C:\Shares\Finance /grant "ACADEMY\GG-Finance-RW:(OI)(CI)M"

04 - Gebruikersomgeving

Teorie: user environment standardizat reduce incidentele repetitive. GPO este mecanismul central de control configuratii user/computer.

Theory: standardized user environment reduces repetitive incidents. GPO is the central mechanism for user/computer configuration control.

New-GPO -Name "Baseline-Users"
New-GPLink -Name "Baseline-Users" -Target "OU=Users,DC=academy,DC=local"
gpupdate /force
gpresult /r

05 - Printerbeheer

Teorie: print management centralizat inseamna mai putine erori de driver, drepturi controlate si deploy simplificat.

Theory: centralized print management means fewer driver issues, controlled rights, and simplified deployment.

Install-WindowsFeature Print-Server -IncludeManagementTools
Add-PrinterPort -Name "IP_192.168.10.50" -PrinterHostAddress "192.168.10.50"
Add-Printer -Name "Office-Printer" -DriverName "Microsoft IPP Class Driver" -PortName "IP_192.168.10.50"

06 - Serverbeheer

Teorie: managementul serverelor implica acces remote securizat, monitorizare continua si reactie bazata pe logs.

Theory: server management requires secure remote access, continuous monitoring, and log-based response.

Enable-NetFirewallRule -DisplayGroup "Remote Desktop"
Get-Service | Where-Object {$_.Status -eq "Running"}
Get-WinEvent -LogName System -MaxEvents 30

07 - Disk Management

Teorie: administrarea stocarii urmeaza flux strict: detectie, initializare, partitionare, formatare, validare.

Theory: storage administration follows strict flow: detect, initialize, partition, format, validate.

Theory: dual-adapter topology separates internet traffic from internal LAN traffic and simplifies educational labs.
