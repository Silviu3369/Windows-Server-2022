Windows Server 2022 - Full course (8 lessons)

Bază: 00-Inleiding, 01-OpzettenDomein, 02-Accounts, 03-File Server, 04-Gebruikersomgeving, 05-Printerbeheer, 06-Serverbeheer, 07-Disk Management.

Based on: 00-Inleiding, 01-OpzettenDomein, 02-Accounts, 03-File Server, 04-Gebruikersomgeving, 05-Printerbeheer, 06-Serverbeheer, 07-Disk Management.

L1. Fundament și arhitectură

Teorie: Windows Server este construit pentru servicii multi-user, centralizare, securitate și management. Înainte de configurări, definești rolurile (DC, file, print, management), standardele de naming și schema logică de rețea.

Theory: Windows Server is designed for multi-user services, centralization, security, and management. Before configuration, define roles (DC, file, print, management), naming standards, and network logic.

Pași / Steps

  1. Definește obiectivul fiecărui server.
  2. Creează nomenclatură (hostname, OU, grupuri).
  3. Stabilește baseline de securitate.

Verificare / Validation

  • Documentul de design e complet.
  • Toți membrii echipei înțeleg topologia.
Get-ComputerInfo | Select WindowsProductName,WindowsVersion
Get-NetIPConfiguration
hostname

L2. Active Directory Domain Services

Teorie: AD DS oferă identitate centralizată, autentificare Kerberos, autorizații și politici. Forest/domain/OUs definesc structura logică a organizației.

Theory: AD DS provides centralized identity, Kerberos authentication, authorization, and policies. Forest/domain/OUs define your organization logic.

Pași / Steps

  1. Instalează AD DS + DNS.
  2. Promovează Domain Controller.
  3. Verifică health-ul domainului.

Verificare / Validation

  • Domain join funcțional.
  • dcdiag fără erori critice.
Install-WindowsFeature AD-Domain-Services,DNS -IncludeManagementTools
Install-ADDSForest -DomainName "academy.local" -DomainNetbiosName "ACADEMY"
Get-ADDomain
dcdiag /v

L3. Accounts, Groups și OUs

Teorie: Identitatea în AD trebuie structurată. Creezi OU-uri pe funcție, grupuri pe rol și aplici delegare minimă pentru operațiuni sigure.

Theory: AD identity must be structured. Build OUs by function, groups by role, and apply minimal delegation for secure operations.

Pași / Steps

  1. Creează OU pe departamente.
  2. Definește grupuri GG/permissions.
  3. Adaugă utilizatori și testează accesul.

Verificare / Validation

  • Obiectele sunt în OU-ul corect.
  • Delegarea funcționează.
New-ADOrganizationalUnit -Name "HR" -Path "DC=academy,DC=local"
New-ADUser -Name "Ana Pop" -SamAccountName apop -Enabled $true -AccountPassword (ConvertTo-SecureString "P@ssw0rd!" -AsPlainText -Force)
New-ADGroup -Name "GG-HR-RW" -GroupScope Global -GroupCategory Security
Add-ADGroupMember -Identity "GG-HR-RW" -Members apop

L4. File Server și securitatea datelor

Teorie: Separi share permissions de NTFS permissions. Folosești grupuri, nu utilizatori direcți, și impui quota pentru controlul consumului.

Theory: Separate share permissions from NTFS permissions. Use groups, not direct users, and enforce quotas for storage control.

Pași / Steps

  1. Instalează rolul File Server.
  2. Creează structura de share-uri.
  3. Aplică AGDLP + quota.

Verificare / Validation

  • Acces controlat per grup.
  • Cota blochează depășirea.
Install-WindowsFeature FS-FileServer,FS-Resource-Manager -IncludeManagementTools
New-Item -Path C:\Shares\Finance -ItemType Directory
New-SmbShare -Name "Finance$" -Path C:\Shares\Finance -FullAccess "Domain Admins"
icacls C:\Shares\Finance /grant "ACADEMY\GG-Finance-RW:(OI)(CI)M"

L5. User Environment și Group Policy

Teorie: GPO standardizează configurațiile pe user/computer. Home folders, scripturi și policy processing trebuie proiectate curat.

Theory: GPO standardizes user/computer configuration. Home folders, scripts, and policy processing must be cleanly designed.

Pași / Steps

  1. Creează GPO baseline.
  2. Link pe OU corect.
  3. Rulează gpupdate/gpresult.

Verificare / Validation

  • Setările apar pe client.
  • Policy precedence e corectă.
New-GPO -Name "Baseline-Users"
New-GPLink -Name "Baseline-Users" -Target "OU=Users,DC=academy,DC=local"
gpupdate /force
gpresult /r

L6. Print Management

Teorie: Print server central simplifică administrarea, drepturile și deployment-ul imprimantelor către grupuri țintă.

Theory: A central print server simplifies administration, permissions, and printer deployment to target groups.

Pași / Steps

  1. Instalează Print Server.
  2. Adaugă port și imprimantă.
  3. Testează job-uri din client.

Verificare / Validation

  • Imprimanta apare pe client.
  • Job-urile se finalizează corect.
Install-WindowsFeature Print-Server -IncludeManagementTools
Add-PrinterPort -Name "IP_192.168.10.50" -PrinterHostAddress "192.168.10.50"
Add-Printer -Name "Office-Printer" -DriverName "Microsoft IPP Class Driver" -PortName "IP_192.168.10.50"
Get-Printer

L7. Server Management și monitorizare

Teorie: Administrarea zilnică include RDP controlat, servicii, procese și audit în Event Viewer pentru incidente.

Theory: Daily administration includes controlled RDP, service/process monitoring, and Event Viewer auditing for incidents.

Pași / Steps

  1. Activează RDP sigur.
  2. Monitorizează servicii critice.
  3. Filtrează event logs relevante.

Verificare / Validation

  • Acces remote funcțional.
  • Evenimentele critice sunt vizibile.
Enable-NetFirewallRule -DisplayGroup "Remote Desktop"
Get-Service | Where-Object {$_.Status -eq "Running"}
Get-WinEvent -LogName System -MaxEvents 25

L8. Disk Management

Teorie: Administrarea stocării include inițializare, partiționare, formatare și validare periodică a volumelor.

Theory: Storage administration includes initialization, partitioning, formatting, and periodic volume validation.

Pași / Steps

  1. Identifică discurile noi.
  2. Initializează și creează volum.
  3. Formatează și etichetează.

Verificare / Validation

  • Volumul e online.
  • Spațiul este raportat corect.
Get-Disk
Initialize-Disk -Number 1 -PartitionStyle GPT
New-Partition -DiskNumber 1 -UseMaximumSize -DriveLetter F
Format-Volume -DriveLetter F -FileSystem NTFS -NewFileSystemLabel "DATA"
