00 - Inleiding

Subcapitol: Windows Client vs Windows Server

Teorie: un sistem Windows Client este optimizat pentru productivitate individuala, in timp ce Windows Server este optimizat pentru servicii multi-user, disponibilitate, securitate centralizata si administrare la scara. In practica, serverul devine sursa de adevar pentru identitate, politici si date partajate.

Theory: Windows Client is optimized for individual productivity, while Windows Server is optimized for multi-user services, availability, centralized security, and scalable administration. In practice, the server becomes the source of truth for identity, policy, and shared data.

Pasi / Steps

  1. Defineste rolurile serverului (DC, File, Print).
  2. Stabileste cine administreaza fiecare rol.
  3. Documenteaza dependentele intre roluri.

Verificare / Validation

  • Rolurile sunt clar separate.
  • Exista schema operationala scrisa.

Subcapitol: Editii, interface type, sistem requirements

Teorie: alegerea editiei (Standard/Datacenter) influenteaza capabilitatile de virtualizare si scalare. Cerintele hardware minime sunt punct de pornire, nu tinta de productie. Pentru stabilitate, planifica resurse peste minim (CPU/RAM/storage) si monitorizare permanenta.

Theory: edition choice (Standard/Datacenter) impacts virtualization and scaling capabilities. Minimum hardware requirements are a starting point, not a production target. For stability, plan above minimum (CPU/RAM/storage) and monitor continuously.

01 - OpzettenDomein (AD DS)

Subcapitol: Structura AD DS

Teorie: AD DS este directory service-ul care organizeaza obiectele IT in structuri logice (forest, domain, OU). Forest-ul defineste limita de securitate superioara. Domeniul gestioneaza identitatea si autentificarea. OU-urile permit delegare si aplicare de politici pe segmente organizationale.

Theory: AD DS is the directory service that organizes IT objects into logical structures (forest, domain, OU). Forest defines the top security boundary. Domain handles identity and authentication. OUs enable delegation and policy application by organizational segment.

Subcapitol: Instalare domeniu pas cu pas

Pasi / Steps

  1. Instaleaza AD DS + DNS role.
  2. Promoveaza serverul in Domain Controller nou.
  3. Configureaza DNS intern ca sursa primara.
  4. Testeaza autentificare domeniu.

Verificare / Validation

  • Get-ADDomain returneaza domeniul corect.
  • dcdiag nu afiseaza erori critice.
  • Clientul poate face domain join.
Install-WindowsFeature AD-Domain-Services,DNS -IncludeManagementTools
Install-ADDSForest -DomainName "academy.local" -DomainNetbiosName "ACADEMY"
Get-ADDomain
dcdiag /v

02 - Accounts

Subcapitol: OU, users, groups

Teorie: managementul identitatii trebuie sa urmeze un model predictibil. Creezi OU-uri pentru structura organizationala, grupuri pentru permisiuni si utilizatori pentru identitate. Evita acordarea de permisiuni direct pe user cand poti folosi grupuri.

Theory: identity management must follow a predictable model. Use OUs for organizational structure, groups for permissions, and users for identity. Avoid direct user permissions when group-based assignment is possible.

New-ADOrganizationalUnit -Name "HR" -Path "DC=academy,DC=local"
New-ADGroup -Name "GG-HR-RW" -GroupScope Global -GroupCategory Security
New-ADUser -Name "Ana Pop" -SamAccountName apop -Enabled $true -AccountPassword (ConvertTo-SecureString "P@ssw0rd!" -AsPlainText -Force)
Add-ADGroupMember -Identity "GG-HR-RW" -Members apop

Subcapitol: Delegare control

Teorie: delegarea permite operationalizare fara a distribui drepturi Domain Admin. Atribui exact taskurile necesare (reset password, create users, manage groups) unui grup operational.

Theory: delegation enables operations without distributing Domain Admin rights. Assign only required tasks (reset password, create users, manage groups) to an operational group.

03 - File Server

Subcapitol: NTFS + Share permissions

Teorie: securitatea pe fisiere are doua straturi: share permissions (acces la share) si NTFS ACL (acces la obiecte). Efectivul final este combinatia lor. In productie, mentii share permissions simple si controlezi granularitatea prin NTFS + grupuri.

Theory: file security has two layers: share permissions (share access) and NTFS ACL (object access). Effective access is their combination. In production, keep share permissions simple and use NTFS + groups for granular control.

Subcapitol: Quota si FSRM

Pasi

  1. Instaleaza FS-Resource-Manager.
  2. Defineste quota templates.
  3. Aplica quota pe foldere tinta.

Verificare

  • Userul nu depaseste limita.
  • Notificarile quota sunt trimise.
Install-WindowsFeature FS-FileServer,FS-Resource-Manager -IncludeManagementTools
New-Item -Path C:\Shares\Finance -ItemType Directory
New-SmbShare -Name "Finance$" -Path C:\Shares\Finance -FullAccess "Domain Admins"
icacls C:\Shares\Finance /grant "ACADEMY\GG-Finance-RW:(OI)(CI)M"

04 - Gebruikersomgeving

Subcapitol: Profiles, home folders, scripts

Teorie: user environment trebuie standardizat pentru predictibilitate operationala. Home folders centralizeaza datele, profilele roaming transfera configurari user, iar logon scripts automatizeaza mapari si setari repetitive.

Theory: user environment must be standardized for operational predictability. Home folders centralize data, roaming profiles carry user settings, and logon scripts automate repetitive mappings and settings.

Subcapitol: Group Policy Management

New-GPO -Name "Baseline-Users"
New-GPLink -Name "Baseline-Users" -Target "OU=Users,DC=academy,DC=local"
gpupdate /force
gpresult /r

Verificare: politicile apar in gpresult si efectul este vizibil pe client.

Validation: policies are visible in gpresult and effects are present on clients.

05 - Printerbeheer

Teorie: print management centralizat reduce costul operational si simplifica suportul. Definesti queue-uri, drivere validate, drepturi pe printer si deployment pe grupuri de utilizatori.

Theory: centralized print management reduces operational cost and simplifies support. Define queues, validated drivers, printer permissions, and group-based deployment.

Install-WindowsFeature Print-Server -IncludeManagementTools

Get-Printer

06 - Serverbeheer

Teorie: server management inseamna operare zilnica controlata: acces remote, monitorizare servicii/procese, audit in logs si reactii rapide la evenimente critice.

Theory: server management means controlled daily operations: remote access, service/process monitoring, log auditing, and fast response to critical events.

Enable-NetFirewallRule -DisplayGroup "Remote Desktop"
Get-Service | Where-Object {$_.Status -eq "Running"}
Get-WinEvent -LogName System -MaxEvents 30

07 - Disk Management

Teorie: storage management corect previne downtime. Fluxul standard este detectare disk -> initialize -> partitionare -> formatare -> validare.

Theory: proper storage management prevents downtime. Standard flow is disk detection -> initialization -> partitioning -> formatting -> validation.

Get-Disk
Initialize-Disk -Number 1 -PartitionStyle GPT
New-Partition -DiskNumber 1 -UseMaximumSize -DriveLetter F
Format-Volume -DriveLetter F -FileSystem NTFS -NewFileSystemLabel "DATA"
Get-Volume

Virtualbox Windows Server Router

Teorie: acest scenariu de laborator simuleaza conectivitatea internet + LAN intern cu 2 adaptoare de retea. Este util pentru medii de invatare unde NAT standard nu functioneaza predictibil.

Theory: this lab scenario simulates internet + internal LAN connectivity using 2 network adapters. It is useful for training environments where standard NAT is not predictable.

Pasi

  1. Configureaza NIC1 pentru internet.
  2. Configureaza NIC2 pentru LAN intern.
  3. Seteaza IP static si DNS pentru domeniu.

Verificare

  • Clientii ating DC-ul.
  • DNS intern raspunde.
Get-NetIPConfiguration
New-NetIPAddress -InterfaceAlias "Ethernet 2" -IPAddress 10.10.10.1 -PrefixLength 24
Test-NetConnection SRV-DC01 -Port 53
