Azure - Curs practic pe 10 capitole

Fiecare capitol urmeaza modelul: teorie -> implementare -> verificare.

Each chapter follows: theory -> implementation -> validation.

1. Inleiding

Teorie: cloud-ul ofera elasticitate, self-service si plata la consum. Diferenta intre IaaS, PaaS, SaaS determina cine administreaza infrastructura.

Theory: cloud provides elasticity, self-service, and pay-as-you-go. IaaS/PaaS/SaaS determines who manages infrastructure.

2. Toegang

Accesul corect incepe cu subscription potrivit si context Azure valid.

Connect-AzAccount
Get-AzSubscription
Set-AzContext -Subscription "My Subscription"

3. Virtuele machines

VM-urile se creeaza in Resource Group si trebuie validate operational imediat dupa deploy.

New-AzResourceGroup -Name rg-lab -Location westeurope
New-AzVM -ResourceGroupName rg-lab -Name vm-lab -Location westeurope
Get-AzVM -Status -ResourceGroupName rg-lab -Name vm-lab

4. Storage

Storage account este containerul principal pentru obiecte de date in Azure.

New-AzStorageAccount -ResourceGroupName rg-lab -Name stlabdemo001 -SkuName Standard_LRS -Location westeurope

5. Netwerking

VNet + subnet + NSG controleaza izolarea si traficul permis.

Get-AzVirtualNetwork
Get-AzNetworkSecurityGroup

6. Webapplicatie

New-AzAppServicePlan -Name asp-lab -Location westeurope -ResourceGroupName rg-lab -Tier Free
New-AzWebApp -Name web-lab-demo-001 -Location westeurope -AppServicePlan asp-lab -ResourceGroupName rg-lab

7. Database

Azure SQL reduce overhead operational fata de SQL instalat manual.

Get-AzSqlServer
Get-AzSqlDatabase

8. Kostenbeheer

Get-AzConsumptionUsageDetail -StartDate (Get-Date).AddDays(-30) -EndDate (Get-Date)

9. Voor wie meer wil

Extinde cu policy, tagging, monitorizare si automation.

10. Multifactor Authentication

Connect-MgGraph -Scopes Policy.Read.All
Get-MgPolicyAuthenticationMethodPolicy
