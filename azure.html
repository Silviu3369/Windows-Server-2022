Azure - Lecții pas cu pas

Conținut inspirat din cursul tău Azure Introductie (2024-12), tradus și ordonat practic.

Content inspired by your Azure Introductie course (2024-12), translated and structured for practice.

L1. Fundamente cloud

Teorie: înțelegi diferența între IaaS, PaaS, SaaS, precum și modelele Public/Private/Hybrid cloud.

Theory: understand IaaS, PaaS, SaaS and Public/Private/Hybrid cloud models.

Pași

  1. Definește nevoile aplicației.
  2. Alege modelul cloud potrivit.
  3. Documentează argumentele tehnice.

Verificare

  • Modelul ales este justificat.
az account show

L2. Acces și subscriptions

Teorie: accesul corect în Azure începe cu subscriptionul potrivit și controlul identității administrative.

Theory: proper Azure access starts with the right subscription and controlled admin identity.

Pași

  1. Conectează-te în Azure.
  2. Setează subscription context.
  3. Verifică accesul în portal/Cloud Shell.

Verificare

  • Subscription corect activă.
Connect-AzAccount
Get-AzSubscription
Set-AzContext -Subscription "My Subscription"

L3. Virtual Machines

Teorie: RG + VM reprezintă fundamentul infrastructurii cloud administrabile.

Theory: RG + VM form the baseline of manageable cloud infrastructure.

Pași

  1. Creează Resource Group.
  2. Provisionează VM.
  3. Testează status și conectivitate.

Verificare

  • VM este running și accesibilă.
New-AzResourceGroup -Name rg-lab -Location westeurope
New-AzVM -ResourceGroupName rg-lab -Name vm-lab -Location westeurope
Get-AzVM -ResourceGroupName rg-lab -Name vm-lab -Status

L4. Storage

Teorie: storage account-ul gestionează datele aplicațiilor (blobs/files/tables/queues) în funcție de scenariu.

Theory: storage accounts manage application data (blobs/files/tables/queues) by scenario.

Pași

  1. Creează storage account.
  2. Configurează container/share.
  3. Testează upload/download.

Verificare

  • Datele sunt accesibile controlat.
New-AzStorageAccount -ResourceGroupName rg-lab -Name stlabdemo001 -SkuName Standard_LRS -Location westeurope
Get-AzStorageAccount -ResourceGroupName rg-lab

L5. Networking și NSG

Teorie: securitatea traficului în Azure se face prin VNet, subnet și NSG rules.

Theory: Azure traffic security is implemented through VNet, subnet, and NSG rules.

Pași

  1. Creează VNet și subnet.
  2. Asociază NSG.
  3. Permite doar traficul necesar.

Verificare

  • Only intended ports are open.
Get-AzVirtualNetwork
Get-AzNetworkSecurityGroup
Get-AzNetworkSecurityRuleConfig -NetworkSecurityGroup (Get-AzNetworkSecurityGroup -Name nsg-lab -ResourceGroupName rg-lab)

L6. Web App

Teorie: App Service permite hosting rapid fără management direct de OS.

Theory: App Service enables fast hosting without direct OS management.

Pași

  1. Creează App Service Plan.
  2. Deploy web app.
  3. Testează endpoint public.

Verificare

  • Site-ul răspunde online.
New-AzAppServicePlan -Name asp-lab -Location westeurope -ResourceGroupName rg-lab -Tier Free
New-AzWebApp -Name web-lab-demo-001 -Location westeurope -AppServicePlan asp-lab -ResourceGroupName rg-lab

L7. SQL Database

Teorie: Azure SQL oferă DB managed pentru aplicații cu cerințe de disponibilitate și securitate.

Theory: Azure SQL provides managed databases for apps requiring availability and security.

Pași

  1. Creează SQL server logic.
  2. Creează baza de date.
  3. Configurează access rules.

Verificare

  • Conexiunea DB este validă.
Get-AzSqlServer
Get-AzSqlDatabase

L8. Cost control și MFA

Teorie: guvernanța Azure cere cost visibility și identitate sigură (MFA + Conditional Access).

Theory: Azure governance requires cost visibility and secure identity (MFA + Conditional Access).

Pași

  1. Analizează consumul lunar.
  2. Setează alerte de buget.
  3. Aplică MFA pe conturile critice.

Verificare

  • Costurile sunt monitorizate.
  • MFA activ pe conturile admin.
Get-AzConsumptionUsageDetail -StartDate (Get-Date).AddDays(-30) -EndDate (Get-Date)
Connect-MgGraph -Scopes Policy.Read.All
Get-MgPolicyAuthenticationMethodPolicy
