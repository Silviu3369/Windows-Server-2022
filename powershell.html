PowerShell - Modul practic

Modul separat cu focus pe productivitate și scripting real de administrator.

Separate module focused on productivity and real admin scripting.

L1. Fundamentals

Teorie: PowerShell lucrează pe obiecte, nu pe text. Acesta este avantajul major în administrare.

Theory: PowerShell works with objects, not raw text. This is a major admin advantage.

Pași

  1. Explorează cmdlet-uri disponibile.
  2. Folosește Help în profunzime.
  3. Rulează comenzi de bază.

Verificare

  • Poți identifica cmdlet-ul potrivit.
Get-Command
Get-Help Get-Service -Full
Get-Process

L2. Pipeline și filtrare

Teorie: pipeline-ul conectează outputul unui cmdlet cu inputul următorului.

Theory: pipeline connects one cmdlet output to the next cmdlet input.

Pași

  1. Filtrează servicii active.
  2. Selectează proprietăți utile.
  3. Sortează după criterii relevante.

Verificare

  • Output-ul este clar și relevant.
Get-Service | Where-Object Status -eq "Running" | Select-Object Name,Status
Get-Process | Sort-Object CPU -Descending | Select-Object -First 10

L3. Scripturi și funcții

Teorie: scripturile standardizează task-uri repetitive și reduc erorile umane.

Theory: scripts standardize repetitive tasks and reduce human errors.

Pași

  1. Creează script cu parametru.
  2. Transformă logica într-o funcție.
  3. Testează cu input diferit.

Verificare

  • Scriptul rulează idempotent.
param([string]$Name="World")
Write-Host "Hello $Name"
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser

L4. Error handling

Teorie: gestionarea erorilor face scriptul robust și sigur în producție.

Theory: proper error handling makes scripts robust and production-safe.

Pași

  1. Folosește try/catch/finally.
  2. Loghează erorile.
  3. Definește ieșire clară pentru fail.

Verificare

  • Erorile sunt capturate și explicate.
try { Get-Item C:\missing.txt -ErrorAction Stop }
catch { $_ | Out-File .\errors.log -Append }
finally { "done" }

L5. Remote administration

Teorie: PSRemoting permite administrare centralizată pe multiple servere.

Theory: PSRemoting enables centralized administration across multiple servers.

Pași

  1. Activează remoting.
  2. Testează comenzi remote.
  3. Folosește sesiuni persistente.

Verificare

  • Comenzile rulează de la distanță fără erori.
Enable-PSRemoting -Force
Invoke-Command -ComputerName SRV-DC01 -ScriptBlock { Get-Service }
Enter-PSSession -ComputerName SRV-DC01

L6. Automation practică

Teorie: automatizarea zilnică (rapoarte, verificări, cleanup) crește consistența operațională.

Theory: daily automation (reports, checks, cleanup) improves operational consistency.

Pași

  1. Scrie scriptul operațional.
  2. Rulează manual testul.
  3. Planifică execuția periodică.

Verificare

  • Task-ul rulează la oră și produce output.
Register-ScheduledTask -TaskName "DailyHealth" -Action (New-ScheduledTaskAction -Execute "powershell.exe" -Argument "-File C:\Scripts\health.ps1") -Trigger (New-ScheduledTaskTrigger -Daily -At 08:00) -User "SYSTEM" -RunLevel Highest
