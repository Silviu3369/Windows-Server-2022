Troubleshooting pas cu pas

Fiecare incident are: simptome, cauza probabila, pasi de diagnostic, remediere, verificare.

1. AD Login Failure

Simptome: user nu poate face login in domeniu. Cauza frecventa: DNS gresit pe client.

Pasi

  1. Verifica DNS client.
  2. Detecteaza DC.
  3. Verifica events lockout.

Verificare

  • User poate face login normal.
ipconfig /all
nltest /dsgetdc:academy.local
Get-WinEvent -LogName Security -MaxEvents 50

2. GPO Not Applying

gpupdate /force
gpresult /h c:\temp\gpo.html

3. Access Denied on Share

whoami /groups
Get-SmbShareAccess -Name Finance$
icacls C:\Shares\Finance

4. Printing Incidents

Get-PrintJob -PrinterName Office-Printer
Restart-Service Spooler
Get-PrinterDriver

5. Azure VM Inaccessible

Get-AzVM -Status -ResourceGroupName rg-lab -Name vm-lab
Get-AzNetworkSecurityGroup
Get-AzPublicIpAddress

6. M365 Sign-In Blocked

Get-MgAuditLogSignIn -Top 20
Get-MgIdentityConditionalAccessPolicy
Get-MgPolicyAuthenticationMethodPolicy
